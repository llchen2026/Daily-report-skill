# 中东能源化工情报与投资观点生成技能

## 目的

自动执行以下完整流程：(1) 读取全球能源化工AI看板的聚合数据与边际更新；(2) 并行执行看板数据验证+关键资产核实+akshare能化期货价格采集；(3) 合并执行区域新闻搜索+政策法规搜索+化工品专项搜索（拿到akshare数据后筛选强势品种）；(4) 按四层置信度分层+四维分析框架对新闻进行标准化、去重清洗与边际变化判定，沉淀为JSONL清洗底座；(5) 生成投资观点看板HTML并截图输出。适用于每日/每班次的能源化工研究与投资观点跟踪。

## 触发条件

当用户提到以下关键词或意图时触发：中东能源看板、原油情报、成品油分析、化工品投资观点、霍尔木兹海峡、地缘冲突原油、能化日报、边际变化追踪、oil intelligence、petrochemical outlook、energy dashboard。

## 执行流程（5步）

### 步骤A：抓取看板聚合数据

使用 WebFetch 抓取以下三个页面的完整数据，提取所有数值指标：

1. **主看板**: `https://middle-east-dashboard-public.pages.dev/`
   - 提取：受影响汇总指标（原油/天然气/LNG/炼能/各化工品产能影响量，kbpd/mtpa/mcm-d）
   - **炼能拆分提取**：将受影响炼能分为两类——"受损"（袭击/爆炸/设施损毁导致的物理损坏）和"不可抗力&检修"（FM声明/计划检修/原料断供/行政关停）
   - 提取：天然气液化受影响量（LNG出口中断mtpa + 管道气损失mcm/d）
   - 提取：霍尔木兹航运摘要（海峡状态、通行船数、滞留油轮、绕行载货估算）
   - 提取：当前边际影响占全球比重
   - 提取：重点原油库存总表（美国/欧洲/日韩/中国台湾/印度/泰国库存水平与可用天数）
   - 提取：欧洲ARA油品库存、欧洲天然气库存、新加坡燃料库存
   - 提取：亚洲天然气月度库存
   - 提取：国家级能化产品短缺整理
   - 提取：重点化工原料状态
   - 提取：近期事件时间线

2. **资产修复进度表**: `https://middle-east-dashboard-public.pages.dev/recovery-progress.html`
   - 提取：上游资产（国家/资产名/状态/产能损失/修复ETA）
   - 提取：中游炼化资产（同上 + 行政干预/修复深度）
   - 提取：下游枢纽（港口/管网/物流状态）
   - 提取：宏观观察因子（LDL-14d / 40天陷阱 / Basra尾迹 / 新战略冲击因子）

3. **航运态势看板**: `https://middle-east-dashboard-public.pages.dev/shipping.html`
   - 提取：顶层概览指标（海峡状态/船舶数/滞留/通行数/吞吐恢复度）
   - 提取：绕行港口离港补偿（Fujairah + Yanbu）
   - 提取：绕行油轮离港明细与持续跟踪

对照 `references/dashboard-structure.md` 确认基线数据结构，将提取的数值作为当前基线快照保存。

**重要原则**：看板抓取的所有聚合数字（如吞吐恢复度56.1%、277艘滞留、布尔甘-2500kbd、炼能受损2200kbpd等）均为**待验证数据点**，不得直接用于投资观点。必须经过步骤B的两层验证后才能使用。未经验证的看板数据只能作为搜索线索。

### 步骤B：并行执行验证搜索 + akshare价格采集

本步骤包含两条**相互独立**的工作流，应尽可能并行执行以节省时间：

---

#### B-1：两层验证 — 看板数据确认 + 关键资产滚动核实

**必须加载** `references/critical-assets-verification.md` 作为核实清单。

看板基线数据可能滞后或失准，投资观点不能直接基于可能过时的基线数字。本步骤执行**两层验证**：

**第一层：看板聚合数据确认（Dashboard Data Cross-Check）**

对步骤A从看板抓取的**每个关键聚合数字**执行独立搜索确认。这些数字是看板的"核心卖点"，但也是最可能失准的：

**必须确认的看板聚合指标**：

| 看板指标 | 搜索确认方式 | 合格标准 |
|----------|-------------|----------|
| 霍尔木兹吞吐恢复度（如56.1%） | 搜索"Hormuz shipping throughput" + 油轮跟踪数据（LSEG/Kpler/IEA） | 至少1个T1来源给出可比量级 |
| 滞留油轮数（如277艘） | 搜索"tankers stranded Persian Gulf" + 船舶跟踪报道 | T1/T2来源确认量级一致 |
| 受影响原油量（如-XXX kbd） | 搜索各P0资产的产量数据汇总对照 | 各资产核实结果加总与看板汇总误差<15% |
| 炼能受损/不可抗力（如2200/1000 kbpd） | 搜索各炼厂状态汇总对照 | 各炼厂核实结果加总与看板汇总误差<15% |
| LNG受影响量（如42.5 mtpa） | 搜索"Qatar LNG export disruption" + QatarEnergy官方声明（能源部长Al-Kaabi确认Trains 4&6受损12.8 MTPA） | T1来源已确认核心受损量，看板口径需拆解为：物理受损12.8 MTPA + Pearl GTL折算 + 物流性停产折算 |
| 国家级短缺标签 | 搜索该国官方声明/运营商FM公告 | 有官方或运营商确认 |

**确认判定**：
- ✅ **CONFIRMED**：至少1个独立T1来源给出可比量级的数字 → 可用于观点
- ⚠️ **PARTIAL**：只有T2/T3来源，或量级偏差>15% → 标注偏差，谨慎使用
- ❌ **UNVERIFIED**：搜索不到任何独立来源确认 → **不得用于投资观点**，改用权威替代数据（见下）

**权威替代数据优先级**（当看板数据UNVERIFIED时）：
1. **IEA月报库存数据**：OECD商业库存天数、全球液体库存变化
2. **EIA周报库存数据**：美国商业原油库存、汽油库存、馏分油库存、Cushing库存
3. **API周报库存数据**：美国原油及成品油库存变化
4. **OPEC月报产量数据**：各成员国产量、减产执行率
5. **JODI数据库**：各国进出口量
6. **Kpler/LSEG油轮跟踪**：实际原油海运量

> 核心原则：**如果看板数字无法double check，宁可不用这个数字。** 改用IEA/EIA/OPEC的权威库存/产量数据作为分析基础。权威库存数据是"硬数据"（实测采样），比看板推算的"软数据"可靠得多。

**第二层：关键资产状态滚动核实（Asset-Level Verification）**

按优先级执行定向搜索：
- **P0（必须核实）**：产能损失 >500 kbd 的上游资产 + "全境熔断/完全停产/完全摧毁"资产 + 霍尔木兹海峡通行状态
- **P1（应当核实）**：产能损失 100-500 kbd 的资产 + 修复ETA ≤30天的资产 + 所有中游炼化P1资产
- **P2（滚动覆盖）**：其余资产视搜索预算决定是否核实

对每个P0/P1资产执行一次定向WebSearch，使用 `critical-assets-verification.md` 中的EN+CN关键词组合，时间窗口放宽到90天。

搜索结果与基线对照，判定为：
- **CONFIRMED**：最新信息与基线一致
- **UPDATED**：资产状态有变化（复产/恶化/修复ETA变更/产能修正），用核实结果覆盖基线
- **STALE**：搜索不到任何近期信息（>30天无更新），标注⚠️基线可能滞后

对于关键产量数字（P0资产），额外执行：
- 搜索运营商/能源部官方声明
- 搜索IEA月报/OPEC月报对该资产的描述
- 多家T1来源引用不同数字时，以最新T1来源为准

核实结果汇入 `asset_verification` 数组，在看板"资产状态核实"板块展示基线vs核实状态，UPDATED和STALE状态高亮。

**成本链路数据搜索**（用于观点生成的成本链路维度）：

在核实资产的同时，搜索以下成本链路相关数据：

| 品类 | 搜索内容 |
|------|---------|
| **原油成本链路** | OPEC财政盈亏平衡油价 / 页岩油盈亏平衡线(Permian) / VLCC+Suezmax运费 / 战争风险保险费率 |
| **成品油成本链路** | 区域裂解价差(Rotterdam/Singapore/Gulf) / 成品油跨区套利窗口 |

> 化工品成本链路搜索推迟到步骤C，因为需要先拿到akshare数据筛选强势品种后再定向搜索。

**两层验证汇总**：第一层确认看板聚合数字是否可信，第二层确认各资产状态是否准确。两层均通过的数据可直接用于观点；任一层不通过的，标注问题并用权威替代数据（IEA/EIA/OPEC库存）替代。

#### B-2：获取能化商品实时价格（akshare）

运行 `scripts/fetch_akshare_data.py`，获取18个能化期货主力行情、裂解价差、EIA库存、成品油调价等。

```bash
python3 <skill-path>/scripts/fetch_akshare_data.py <output-dir>
```

该脚本通过akshare获取：
- **国内能化期货主力行情**（18个品种）：INE原油(SC)、燃料油(FU)、低硫燃料油(LU)、沥青(BU)、聚丙烯(PP)、线性PE(L)、苯乙烯(EB)、乙二醇(EG)、PTA(TA)、对二甲苯(PX)、瓶片(PR)、甲醇(MA)、PVC(V)、纯碱(SA)、LPG(PG)、短纤(SP)、橡胶(RU)、20号胶(NR)
- **裂解价差**：各品种相对SC原油的吨级价差（粗略换算）
- **成品油调价历史**：汽油/柴油最新调价（`energy_oil_hist`）
- **EIA原油库存**：最新一期变化值（`macro_usa_eia_crude_rate`）
- **中国能化指数**：日度涨跌（`macro_china_energy_index`）

输出为 `.akshare-data.json`，供步骤C强势品种筛选和步骤E观点生成时引用。

**品种产业链映射**（用于成本链路分析）：
- 上游原油：SC
- 中游炼化：FU, LU, BU, PG
- 烯烃链：PP, L, EG → 下游 EB, PR, SP
- 芳烃链：PX → TA → PR
- 醇类链：MA（煤制/天然气制）
- 氯碱链：SA → V
- 橡胶链：RU, NR

### 步骤C：合并新闻搜索（区域新闻 + 政策法规 + 化工品专项）

本步骤在步骤B完成后执行（需要akshare结果用于化工品品种筛选）。包含三组搜索，应尽量并行：

---

#### C-1：7大区域新闻搜索（过去32小时）

按照 `references/news-search-queries.md` 的查询模板，对以下7大区域/品类执行 WebSearch：

1. **原油**：全球基准(Brent/WTI/OPEC)、中东原油(沙特/伊朗/伊拉克)、俄罗斯原油
2. **成品油**：全球成品油价格与裂解价差、区域炼化利润(新加坡/鹿特丹/美湾)
3. **霍尔木兹海峡**：海峡通行状态、油轮事件、绕行动态
4. **地缘冲突**：以色列-伊朗、红海/胡塞武装、更广泛中东地缘
5. **美国经济指标**：CPI/就业/EIA库存周报/美联储利率
6. **全球重点国家库存**：美国SPR/Cushing、欧洲ARA+天然气、新加坡+日韩+印度、中国
7. **重点化工品**：烯烃链(乙烯/丙烯/丁二烯)、芳烃/醇类(甲醇/醋酸)、聚氨酯/氯碱(MDI/TDI/PVC)

每个搜索使用 `query_keyword_groups` 参数，覆盖中英文双语关键词。

#### C-2：政策法规强制搜索

**任何在区域新闻中出现的政策/法规/制裁/行政决策信号，以及分析中需要引用的政策面内容，都必须在此环节补充搜索确认。严禁依赖模型内置知识直接生成政策分析。**

搜索范围至少覆盖：
- **OPEC+产量决策**：最新的OPEC+部长级会议声明、自愿减产协议执行情况、补偿减产计划
- **国际制裁动态**：美国OFAC最新制裁清单更新、欧盟对俄/对伊制裁包、G7俄罗斯原油价格上限机制执行状态
- **中国能源政策**：成品油调价窗口最新动态、独立炼厂原油进口配额发放、战略储备投放/回补
- **区域气候/能源法规**：欧盟CBAM碳边境调节机制最新进展、美国生物燃料RVO义务标准更新

**有法规编号/文件号的，必须用WebFetch获取法规原文核心条款**，不得仅凭新闻摘要推断。

执行标准：
1. **搜索法规原文**：用WebSearch搜索官方发布文本（如发改委文件号、OFAC公告编号、欧盟OJ引用、OPEC+声明），找到后用WebFetch获取核心条款
2. **搜索配套新闻**：搜索该政策的最新执行动态和市场解读（至少1个T1 + 1个T2来源）
3. **核对内置参考**：与 `references/` 中的内置文档对照，若发现差异以最新搜索的法规原文为准并标注

输出要求：在观点的[政策面]维度中，必须引用法规名称/文件号+核心条款摘要+搜索来源URL。

例外：references目录中已内置的法规文件（如 `china-oil-pricing-mechanism.md`）可直接引用条款，但每次执行仍需搜索确认是否有最新修订。

---

#### C-3：化工品强势品种专项搜索（唯一化工品搜索环节）

**本环节是化工品信息的唯一搜索入口**，前序步骤不再单独搜索化工品。

**第一步：筛选强势品种**

基于步骤B-2的akshare数据 + 步骤A的看板化工原料状态 + 步骤C-1的区域化工品新闻信号，筛选出2-3个当前最强势品种。

强势品种筛选标准（需同时满足至少2条）：
1. akshare日涨幅 ≥2% 或近5日累计涨幅 ≥5%
2. 有装置级边际事件（FM/检修/永久关闭/复产延迟），且该状态已经过步骤B核实
3. 期货价格突破20日均线或基差显著走强

**第二步：对每个入选品种执行四维定向搜索**

化工品分析涉及的原料/装置/供需/需求四个维度都必须通过定向搜索获取最新信息。看板记录的化工原料状态（运营商FM声明/官方公告/行业媒体报道）有明确信息来源，本身就是T1/T2置信度来源，搜索校验时直接引用并与看板交叉对照。

每个入选品种执行以下4组搜索：

| 搜索维度 | 查询模板（以PP为例） | 目的 |
|----------|---------------------|------|
| **装置级状态** | `"PP polypropylene plant force majeure outage 2026"` / `"聚丙烯 装置 停车 检修 2026"` | 确认当前装置开停状态、FM声明、检修计划 |
| **上游原料** | `"naphtha propane propylene feedstock tightness 2026"` / `"石脑油 丙烷 丙烯 原料 紧张 2026"` | 确认上游原料供应是否紧张、裂解装置开工率 |
| **品种供需平衡** | `"polypropylene Asia supply demand balance inventory 2026"` / `"聚丙烯 供需 库存 开工率 2026"` | 确认品种级库存/开工率/进出口/价差 |
| **下游需求** | `"PE/PP film injection molding demand China 2026"` / `"聚烯烃 包装 建材 需求 2026"` | 确认下游消费端景气度 |

上游原料覆盖范围（按产业链）：
- 烯烃链原料：石脑油（约60%海运经霍尔木兹）、丙烷/LPG（PDH原料）、乙烷
- 芳烃链原料：MX/二甲苯
- 醇类链原料：煤炭、天然气

裂解装置动态覆盖范围：
- 韩国：Lotte/LG Chem NCC开工率
- 日本：Mitsubishi/Sumitomo cracker turnaround
- 中国：独立炼厂/民营大炼化（恒力/盛虹/浙石化/荣盛）石脑油采购与开工率
- 中东：SABIC/SPDC ethane cracker status

下游需求覆盖范围：
- 聚烯烃：包装/建材需求
- 聚酯链：纺织/瓶片出口
- 聚氨酯：冰箱/汽车需求
- 氯碱：地产竣工端

**第三步：品种间关联性分析**

当多个入选品种属于同一产业链，必须在分析中标注它们的**产业链传导路径**：

```
烯烃链:  石脑油/丙烷 → 裂解/PDH → 乙烯/丙烯 → PE(线性) / PP(聚丙烯)
                                                  ↓
                                              PR(瓶片/聚酯)← PX → TA → PR
芳烃链:  石脑油/二甲苯 → 重整 → PX → PTA → 聚酯纤维/瓶片
聚氨酯链: 苯/甲苯 → 苯胺/TDA → MDI/TDI → 聚氨酯(PU)
氯碱链:  盐+电 → 烧碱+氯气 → PVC / 纯碱(SA) → 玻璃
```

**关键关联性规则**：
- PP和MDI/TDI分属不同产业链（烯烃 vs 芳烃），不能混为一谈
- 但万华烟台基地同时拥有MDI/TDI装置和75万吨/年PDH→PP装置，基地级FM可能同时影响两条链
- 万华3月7日FM是**交付端FM**（霍尔木兹航运中断导致无法向中东交付），**不是生产端停产**，两套裂解装置（220万吨/年乙烯）仍在高负荷运行
- PP走强的核心驱动是**石脑油供应链断裂**（约60%亚洲海运石脑油经霍尔木兹）→韩国裂解降至~66%→丙烯紧张→PP成本推升，而非万华MDI/TDI的FM
- PR（瓶片）走强与PP部分关联：PR属聚酯链(PX→TA→PR)，但聚烯烃链整体走强会带动市场情绪

**装置核实要求**：化工品涉及的装置（万华FM、乐天FM、Vynova Beek关闭等）必须经步骤B核实后才能作为观点支撑。未经核实的装置状态不得引用。

### 步骤D：新闻标准化、去重清洗与置信度分层

**必须加载** `references/news-standardization-schema.md` 作为标准化规范。

#### D-1 新闻标准化

将步骤B和步骤C的所有搜索结果转为标准化记录，包含以下核心字段：
- `event_id` / `dedup_hash` / `title` / `summary` / `publish_time` / `source_url`
- `source_name` / `source_tier`（T1权威 / T2行业 / T3区域 / T4社交）
- `confidence`（confirmed / likely / unconfirmed / speculative）
- `region` / `category` / `entities` / `event_type`
- `marginal_type`（new_edge / confirms_baseline / no_change）
- `analysis_dims`（四维分析标注）

#### D-2 四层置信度分层

| 层级 | 代码 | 判定标准 | 示例来源 |
|------|------|----------|----------|
| **T1 权威** | `T1` | 通讯社/官方统计/运营商公告 | Reuters, Bloomberg, EIA, IEA, OPEC, 官方声明 |
| **T2 行业** | `T2` | 行业专业媒体 | Argus, Platts, OilPrice, WorldOil, ICIS, RBN Energy |
| **T3 区域** | `T3` | 区域性新闻/财经媒体 | CNBC, Livemint, TimesKuwait, MarineInsight |
| **T4 社交** | `T4` | 社交媒体/开源情报 | Telegram, X/Twitter, United24Media, OSINT账号 |

**事件置信度规则**：
- `confirmed`：T1来源确认 / 运营商官方公告 / 现场影像证据
- `likely`：T2来源交叉验证 / 单一T1来源但证据链较完整
- `unconfirmed`：仅T3-T4来源 / 缺少独立交叉验证
- `speculative`：分析师推测 / "可能"/"预计"

**观点生成规则**：投资观点只能基于 `confirmed` 或 `likely` 置信度的事件。`unconfirmed` 和 `speculative` 事件仅作为情绪面参考信号。

#### D-3 四维分析框架

对每条标准化新闻标注四个分析维度的影响方向：

| 维度 | 定义 | 关键信号 |
|------|------|----------|
| **基本面 (Fundamental)** | 供需平衡、产能利用率、库存水平 | 供应缺口、装置开工率、库存可用天数、OPEC+产量 |
| **政策面 (Policy)** | 政府/OPEC/运营商的行政性决策 | OPEC+增产/减产、出口禁令、制裁、燃料优先指令、利率路径 |
| **情绪面 (Sentiment)** | 市场预期、风险溢价、资金流向 | Polymarket概率、风险溢价、CFTC持仓、期权偏度、新闻密度 |
| **成本链路 (Cost Chain)** | 边际供应成本的结构性变化 | **原油**：OPEC边际开采成本/页岩油盈亏平衡线/海洋运输费率/保险费率升水。**成品油**：原油→炼化加工费→成品油传导。**化工品**：石脑油/乙烷→烯烃→聚合物全链路成本传导 |

每条新闻的 `analysis_dims` 字段需标注：
```json
{
    "fundamental":    {"direction": "bullish|bearish|neutral", "impact": "high|medium|low", "note": "..."},
    "policy":         {"direction": "bullish|bearish|neutral", "impact": "high|medium|low", "note": "..."},
    "sentiment":      {"direction": "bullish|bearish|neutral", "impact": "high|medium|low", "note": "..."},
    "cost_chain":     {"direction": "bullish|bearish|neutral", "impact": "high|medium|low", "note": "..."}
}
```

#### D-4 去重与边际变化判定

1. **去重**：使用 `scripts/deduplicate_news.py` 对标准化记录执行去重。哈希基于 `实体集合+事件类型+数值变化指标+日期`。
2. **时间过滤**：只保留发布时间在过去32小时内的新闻。
3. **边际变化判定**：将每条炼化/装置相关新闻与 `references/dashboard-structure.md` 中的基线表对照：
   - **新边际变化** (`new_edge`)：资产状态变更、产能损失变化、修复ETA变化、新增资产、新FM声明
   - **确认旧状态** (`confirms_baseline`)：新闻仅确认看板已有信息
   - **无变化** (`no_change`)：与基线完全一致
4. **关联性过滤**：排除与能源化工无关的纯政治新闻。

#### D-5 新闻沉淀

将去重后的标准化记录写入 JSONL 清洗底座：

```bash
python3 <skill-path>/scripts/deduplicate_news.py <input.json> <output-dir>/news-store/
```

沉淀文件位置：`<output-dir>/news-store/YYYY-MM.jsonl`（按月归档，JSONL格式，每行一条记录）。

写入前自动读取近2个月的已有哈希集合，跳过重复记录。沉淀后的数据用于：
- 后续执行的边际追踪基线（读取近30天记录构建"资产→最新状态"映射）
- 去重清洗底座（跨执行周期的去重）
- 历史信号密度分析（情绪面）

## 关键规则

1. **单位口径统一**：原油=kbpd，天然气=mcm/d，LNG=mtpa，化工品=mtpa。炼能必须拆分为"受损"和"不可抗力&检修"两个口径，分别统计。akshare价格单位：SC=元/桶，其他=元/吨。搜索到的新闻数据需折算为一致单位后再与基线对照。

2. **涨跌颜色**：遵循中国习惯，涨用红色(`#ef4444`)，跌用绿色(`#22c55e`)。

3. **基线优先 + 两层验证防护**：所有边际变化判定必须先加载 `references/dashboard-structure.md` 作为基线表。看板数据（包括聚合指标和资产状态）可能过时或失准，所有关键数字必须经过步骤B的两层验证后才能用于观点生成：
   - **第一层（看板聚合确认）**：看板的核心聚合数字必须通过独立搜索double check。确认通过标✅，部分确认标⚠️，未确认标❌。
   - **第二层（资产级核实）**：各P0/P1资产状态逐项搜索核实。CONFIRMED/UPDATED/STALE。
   - **无法double check的数据不用**：看板聚合数字如果搜索不到任何独立来源确认（UNVERIFIED），不得用于投资观点。改用IEA/EIA/OPEC/JODI/Kpler等权威数据替代。

4. **置信度门槛**：投资观点只能基于 `confirmed` 或 `likely` 置信度的事件。`unconfirmed` 和 `speculative` 仅作为情绪面参考。

5. **时区**：所有时间标注为北京时间(GMT+8)。

6. **可靠性声明**：生成的看板底部必须保留"不构成投资建议"声明。

7. **新闻不追溯**：只处理过去32小时窗口内的新信息，不做历史回溯分析（历史已在看板基线中）。

8. **新闻沉淀**：每次执行必须将标准化新闻写入JSONL清洗底座，为后续执行提供去重和边际追踪基线。

9. **四维分析**：每条新闻和每个投资观点必须覆盖基本面/政策面/情绪面/成本链路四个维度，不允许遗漏。

10. **关键信号定义**：关键信号板块只展示看板数据与新闻边际叠加的信号——即新闻提供了对看板已有数据/状态的增量信息。纯看板静态内容（如"霍尔木兹海峡限制通行"如果没有新的新闻边际）不得单独作为信号出现。看板是分析的养料，不是独立展示项。

11. **化工品分化 + 信息搜索全覆盖**：化工品种间存在显著分化，不允许统一分析。每次执行使用步骤C-3筛选2-3个强势品种独立分析。化工品分析涉及的所有维度——上游原料、中游裂解装置、品种级供需平衡、下游需求——**都必须通过定向搜索获取最新信息**，严禁仅凭akshare价格走势和模型内置知识推断化工产业链逻辑。看板记录的化工原料状态（运营商公告/官方声明/行业媒体）本身即为T1/T2置信度来源，搜索校验时直接引用并与看板交叉对照。

12. **中国成品油定价 + 政策法规强制搜索**：
    - **中国成品油定价机制**：中国汽柴油实行公式化联动（Brent+Dubai+Minas十日移动均价→10工作日调价→50元/吨门槛→40/130管控区间）。分析中**禁止使用**"调价上涨预期""政策面推动调价""发改委调价意图"等主观表述。必须加载 `references/china-oil-pricing-mechanism.md`。
    - **政策法规强制搜索**：所有涉及政策、法规、制裁、行政决策的内容，**严禁依赖模型内置知识直接生成分析**。当分析涉及OPEC+产量决策、国际制裁(OFAC/欧盟/G7)、国家级能源政策、中国能源政策、区域气候法规(CBAM/RVO/REPowerEU)、税收关税、任何[政策面]维度时，必须执行：①搜索法规原文（有文件号必须WebFetch原文）②搜索配套新闻（至少1个T1+1个T2来源）③对照内置references文档，若发现差异以最新搜索的法规原文为准。在观点的[政策面]维度中必须引用法规名称/文件号+核心条款摘要+搜索来源URL。
